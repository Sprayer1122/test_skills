| Method | Command | Use Case |
|--------|---------|----------|

| **etautotest Single** | `bin/etautotest -l` | Single testcase execution |

| **Cron/Nightly** | Scheduled scripts | Automated nightly runs |

- `bin/etautotest` - Main orchestrator script (Perl)

### Environment Variables

| Variable | Purpose | Set By |
|----------|---------|--------|
| `$TOP` | Infrastructure root directory | etautotest |
| `$INFRADIR` | Same as TOP | etautotest |
| `$PLATFORM` | Platform identifier (lnx86, sun4v) | bin/try.pl |
| `$GOLD` | Gold directory suffix | Derived from platform |
| `$MODUS_HOME` | Modus installation path | setmodus |
| `$AREA` | Scratch area for workdirs | Environment |
| `$MODUS` | TRUE/FALSE - scratch vs local workdir | Environment |

### Key Files

- `bin/try.pl` - Platform detection
- `bin/lic_validate` - License validation
- `bin/license_test.pl` - License testing
- `setmodus` - Modus environment setup (`/home/et/tools/setmodus`)

### Agent Debugging

```bash
# Check if Modus is invoked
tbver

# Check platform detection
perl bin/try.pl

# Check license status
lmstat -a | grep Modus
```

### Control File Logic

| File | Effect |
|------|--------|
| `IGNORE` | Skip testcase entirely |
| `IGNORE.<FEATURE>` | Skip for specific feature |
| `SKIP_lnx86` | Skip on Linux x86 |
| `SKIP_sun4v` | Skip on Solaris |
| `ON_lnx86` | Run ONLY on Linux x86 (skip others) |
| `ON_sun4v` | Run ONLY on Solaris (skip others) |


|------|---------|--------|

| 1-4 | Warning | Log warning, continue |
| 5+ | Error | Mark FAIL, possibly stop |
| 128+ | Signal/Crash | Mark FAIL, check core dump |

### Key Scripts

- `bin/cdsDiff.pl` - Main diff engine

- `bin/check_tolerance.pl` - Tolerance checker (default Â±5%)

- `filter/<command>` - Command-specific filters

### Comparison Command Pattern in test.log

The actual comparison uses `cdsDiff.pl` with platform detection (general pattern):
```bash
# Status comparison
perl $TOP/bin/cdsDiff.pl -f $TOP/filter/global_filter \
  ./golds.`perl $TOP/bin/try.pl`/status.log ./status.log > ./status.diff

# Command log comparison
perl $TOP/bin/cdsDiff.pl -f $TOP/filter/<command> \
  ./golds.`perl $TOP/bin/try.pl`/log_<command> \
  ./testresults/logs/log_<command> > ./<command>.diff
```

**Note:** `$TOP` refers to the infrastructure tools directory (e.g., `/lan/fed/etpv/src/tools`).

### Diff Cleanup Behavior

After all comparisons, `remove_diff.pl` is called:
- **Empty diff.bak files (0 bytes)** are removed (comparison passed)
- **Non-empty diff.bak files** are preserved (comparison failed)
- If **all diff.bak files are removed** â†’ testcase PASSED
- If **any diff.bak file remains** â†’ testcase FAILED

**Successful comparison output (example):**
```
Removing 0 byte file: <command1>.diff.bak
Removing 0 byte file: <command2>.diff.bak
Removing 0 byte file: status.diff.bak
```

**PASSED testcase cleanup:**
```
********************************************************
**  Testcase passed, removing tbdata and testresults  **
********************************************************
```

## Agent Debugging Guide by Phase

### Phase-by-Phase Checklist

| Phase | Issue | Files to Check | Commands |
|-------|-------|----------------|----------|
| **1-Invocation** | Script not found | `bin/etautotest` | `which etautotest` |
| **2-Environment** | Modus not found | Environment | `tbver`, `echo $MODUS_HOME` |
| **2-Environment** | License failure | `lic.debug.log`, `license_report.log` | `lmstat -a` |
| **3-PreCheck** | Test skipped | `IGNORE*`, `SKIP_*`, `ON_*` | `ls -la IGNORE* SKIP_* ON_*` |
| **4-Workspace** | Permission denied | Directory | `ls -la`, `whoami` |
| **5-Execute** | Command failed | `status.log`, `testresults/logs/log_<cmd>` | `cat status.log` |
| **5-Execute** | Core dump | `core.*` files | `ls core.*`, `gdb modus core.*` |
| **7-Verify** | Diff found | `*.diff.bak` | `cat <cmd>.diff.bak` |
| **7-Verify** | No gold | `golds.linux*` | `ls golds.linux*` |
| **9-Final** | Test killed | `test.log` | `grep Interrupted test.log` |

### Quick Diagnostic Commands

```bash
# 1. Full status check (start here)
cat test.out

# 2. Framework summary
cat autoTest.log

# 3. Check all exit statuses
cat status.log

# 4. Find all failures (remaining diff.bak = failures)
ls -la *.diff.bak 2>/dev/null

# 5. Check specific command exit
grep <command> status.log

# 6. View command log
cat testresults/logs/log_<command>

# 7. Find correct gold directory
ls golds.linux*

# 8. Compare gold vs actual (find gold location first)
cat golds.linux26_64/status.log
diff golds.linux26_64/log_<cmd> testresults/logs/log_<cmd>

# 9. Check control files
ls -la IGNORE* SKIP_* ON_* 2>/dev/null

# 10. Check testcase metadata
cat readme.tsf | head -40

# 11. Check historical status
tail -30 testcase.history

# 12. View diff backup content
cat status.diff.bak
cat <command>.diff.bak

# 13. Check runtime statistics
cat runtime_statistics

# 14. Reproduce in modus (use runme0.tcl for full flow)
modus -f runme0.tcl
```

### Understanding Failure Types from test.out

| test.out Pattern | Meaning | Next Steps |
|------------------|---------|------------|
| `PASSED` | Test passed | No action needed |
| `Exit status diff in <cmd>` | Command exit code changed | Check `status.diff.bak`, compare `status.log` to gold |
| `Diff in <cmd>` | Output content differs | Check `<cmd>.diff.bak`, compare logs |
| `Core Dump in <cmd>` | Command crashed | Check `core.*`, analyze with debugger |

### Understanding status.log Format

```
EXIT STATUS for build_model is 5
 
EXIT STATUS for build_testmode is 5
 
EXIT STATUS for verify_11491_boundary is 5
```

**Note:** Exit status 5 is very common for Modus commands. The important thing is whether it **matches** the gold value.

## Key Decision Points Summary

| Decision | Condition | Result |
|----------|-----------|--------|
| IGNORE file exists? | Yes | SKIP testcase |
| SKIP_<PLATFORM> exists? | Yes | SKIP on that platform |
| ON_* files exist but not ON_<PLATFORM>? | Yes | SKIP (wrong platform) |
| License available? | No | WAIT & RETRY or FAIL |
| Exit status 0? | Yes | PASS command |
| Exit status 1-4? | Yes | WARNING, check tolerance |
| Exit status 5+? | Yes | ERROR, mark FAIL |
| Exit status 128+? | Yes | SIGNAL/CRASH, check core |
| Diff empty? | Yes | PASS verification |
| Diff within tolerance? | Yes | PASS with note |
| Real diff? | Yes | FAIL verification |
